setSubLevelItem()

MenuTrait.php : 145

Set a sub menu item.

public static setSubLevelItem(string $sMainMenu, array<string|int, mixed> $aSubmenu) : bool

array structure: type => if type is given, use special display. only one type exists for now, 'separator', which results in a line separation in the menu. if type is set the only other parameter is position. leave type out for linked menu items, ie. it is assumed that their type is 'link' module => module to load (used for link) action => action to load (used for link, optional) title => slug, unique identifier icon => icon class, must contain leading fa(l|s|r) label => navigation text position => integer defining the position, submenu will be sorted by position ASC before the array is returned. if left empty 0 will be assumed and then sorted by title right => only users with this right will see this menu item role => only users with this role will see this menu item.

Right overwrites role. if both are set, the user role will be checked. if the user is still in the "run" with his role, it will be checked if he has the right mentioned. Technically speaking there should be not too many reasons to have both parameters.

If the parent menu (for instance the settings menu) is ruled out by user role all submenu items will be removed.

setToplevelItem()

MenuTrait.php : 87

Set a top level item for the menu. This should be used scarce to not bloat the top navigation. If a module is defining a menu item, it should also be the only module that provides functionality to this menu.

private static setToplevelItem(string $menuSlug[, array<string|int, mixed> $mainMenu = [] ]) : array<string|int, mixed>

Top level items do not contain an url and are only for opening a submenu.

Array structure: label => Menu label (containing the icon if needed) title => Title slug, used for ids and structuring submenu => array containing the submenu position => integer defining the position, array will be sorted by position ASC before the array is returned. right => only users with this right will see this menu item role => only users with this role will see this menu item.

Right overwrites role. if both are set, the user role will be checked. if the user is still in the "run" with his role, it will be checked if he has the right mentioned. Technically speaking there should be not too many reasons to have both parameters.
