Prices2.php : 83

Manage and calculate prices for tours Prices are structured from a global price setup to agency group prices and agency prices. The setup is forgiving if no prices are added on the lower levels, thereby loading prices from the next higher level. If no price is set for an agency the agencygroup price setup is used, if this is not found then the global price setup.

The only required price per seat type is the global retail price. We offer NP (net prices) and RP (retail prices) with NPs being the prices offered to agencies and operators - used in invoices to agencies. RPs are the price the end customer pays. On vouchers and bills the RP is shown, on tax invoices and receipts the NP. A price object is based on the seat type with the following structure: Object:

  • id (autoincremental, unique, unused)
  • type (\Booka\Tours\PriceTypes::MAIN, ::AGENCY or ::AGENCYGROUP)
  • typeid (if not ::MAIN then add id of agency or agencygroup)
  • seatid
  • offset (if tiered price setup is used then this is the offset of the trip date to the booking date)
  • rp_adult (retail price per adult)
  • np_adult (net price per adult)
  • rp_child (retail price per child)
  • np_child (net price per child)
  • rp_infant (retail price per infant)
  • np_infant (net price per infant) Tiered price setup: If a tiered price setup is used then at least one rp_adult is needed. Further items of the array contain an offset and more prices. When loaded, the array is sorted by offset (default price has offset of 9999 days and is fallback price. Prices are structured as follows:
  • is_deductable = price will be subject to commission
  • is_extra = calculated separate addon items
  • is_separated = listed separatedly on invoices
  • is_taxable = exactly that, taxable items or not
